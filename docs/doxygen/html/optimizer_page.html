Optimizer Reference

Documentation for the optimizer class and training loop.

Introduction

The optimizer class manages the training loop for machine learning models, including forward/backward passes, gradient updates, and checkpoint management.

Training Workflow

The optimizer executes the following workflow per epoch:

  1. Training Loop: Forward pass → Loss → Backward pass → Update
  2. Validation Loop: Forward pass → Metrics (no gradients)
  3. Checkpoint: Save model state if improved

Optimizer Parameters

The optimizer_params_t structure configures training:

General Parameters

Property Type Description
optimizer string Optimizer type ("Adam", "SGD", etc.)
lr double Learning rate
weight_decay double L2 regularization
eps double Epsilon for numerical stability

Adam-specific Parameters

Property Type Description
betas tuple<float, float> Beta1, Beta2 for Adam
amsgrad bool Use AMSGrad variant

SGD-specific Parameters

Property Type Description
momentum double Momentum factor
dampening double Dampening for momentum
nesterov bool Use Nesterov momentum

LBFGS-specific Parameters

Property Type Description
max_iter int Maximum iterations per step
max_eval int Maximum function evaluations
tolerance_grad double Gradient tolerance
tolerance_change double Change tolerance
history_size int History size for L-BFGS

Example

optimizer_params_t params;
params.optimizer = "Adam";
params.lr = 0.001;
params.weight_decay = 1e-5;
params.betas = {0.9, 0.999};
params.amsgrad = true;
optimizer opt;
opt.m_settings = settings;
opt.import_dataloader(&dl);
opt.import_model_sessions(&model_params);
for (int epoch = 0; epoch < 100; ++epoch) {
for (int k = 0; k < 5; ++k) {
opt.training_loop(k, epoch);
opt.validation_loop(k, epoch);
}
}

Event Template Guide

Detailed guide for using and extending event_template.

Introduction

The event_template class is the base class for all event data structures in AnalysisG. It provides the interface for reading physics data from ROOT files and building structured event representations.

Components

An event template manages three levels of data organization:

  1. Trees: ROOT TTree objects containing the data
  2. Branches: Collections of related variables (e.g., "Electrons")
  3. Leaves: Individual variables (e.g., "Electrons.pt")

Example

Here's how to create a custom event template:

class MyEvent : public event_template {
public:
MyEvent() {
// Define which tree to read
add_tree("nominal");
// Define branches (particle collections)
add_branch("electrons", "el_");
add_branch("muons", "mu_");
add_branch("jets", "jet_");
// Define individual leaves
add_leaf("met", "met_et");
add_leaf("met_phi", "met_phi");
// Register particle types
register_particle<Electron>("electrons");
register_particle<Muon>("muons");
register_particle<Jet>("jets");
}
void build(element_t* el) override {
// Custom event building logic
// Access raw data via el->data
}
};

Properties

Event templates expose several properties:

Property Type Description
name string Template identifier
hash string Unique configuration hash
tree string Primary TTree name
weight double Event weight
index long Event index in source
leaves vector<string> List of leaf names

Particle Management

Particles are managed through:

  • register_particle(): Register a particle type
  • particle_link: Access registered particles by collection
  • particle_generators: Factory functions for particle creation

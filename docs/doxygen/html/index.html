AnalysisG Framework Documentation

Introduction

AnalysisG is a comprehensive framework designed for graph-based analysis in high-energy physics (HEP). It provides a flexible and extensible architecture for processing physics event data, constructing graph representations, applying machine learning models, and evaluating analysis results.

The framework is built with modern C++ and integrates with PyTorch for machine learning capabilities, making it suitable for both traditional physics analysis and cutting-edge ML-based approaches.

Architecture Overview

The AnalysisG framework is organized into several main components:

  • Core Module (Core Module): Contains the foundational C++ implementations for graph structures, nodes, and edges that form the basis of the graph-based analysis approach.
  • Modules (Modules): Contains specialized modules that extend the framework's functionality, including event processing, graph building, model training, and metrics evaluation.
  • CUDA Acceleration (CUDA Accelerated Computing): GPU-accelerated physics computations and tensor operations for high-performance analysis.
  • Python Bindings (Cython Python Bindings): Cython-based Python interface for seamless integration with Python workflows and Jupyter notebooks.
  • Pre-built Templates (Pre-built Templates): Ready-to-use event, graph, model, and selection templates for common HEP analysis tasks.

Getting Started

To begin using AnalysisG, you'll typically:

  1. Define your event data structure by subclassing event_template
  2. Create particle representations using particle_template
  3. Build graph structures with graph_template
  4. Apply selections using selection_template
  5. Train or apply models using model_template
  6. Evaluate results with metric_template

See Getting Started with AnalysisG for a detailed guide.

Python Interface

The framework provides comprehensive Python bindings via Cython:

from AnalysisG.core import Analysis
ana = Analysis()
ana.OutputPath = "results/"
ana.AddSamples("/data/", "sample")
ana.Start()

See Cython Python Bindings for Python API documentation.

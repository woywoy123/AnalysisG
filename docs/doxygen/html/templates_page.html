Template Classes

Documentation for the core template classes in AnalysisG.

Introduction

AnalysisG uses a template-based design pattern where base template classes define the interface and common functionality, while user-defined subclasses implement analysis-specific logic.

Tools Base Class

The tools class provides common utility functions inherited by most templates:

  • String manipulation
  • File path handling
  • Hash computation
  • Timing utilities

Notification Base Class

The notification class provides logging functionality:

  • Message logging at various levels
  • Progress reporting
  • Warning and error tracking

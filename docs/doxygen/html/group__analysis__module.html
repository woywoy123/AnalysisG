Overview

The Analysis module contains the main analysis class, which serves as the central orchestrator for the entire analysis workflow. It coordinates:

  • Data loading and sample management
  • Event processing and reconstruction
  • Selection application
  • Graph construction
  • Model training and inference
  • Metric evaluation

Workflow

A typical analysis workflow using the analysis class:

  1. Create an analysis instance
  2. Add data samples with add_samples()
  3. Add event templates with add_event_template()
  4. Add graph templates with add_graph_template()
  5. Add selection templates with add_selection_template()
  6. Add models with add_model()
  7. Add metrics with add_metric_template()
  8. Configure settings via m_settings
  9. Start execution with start()

Configuration

The analysis class uses the settings_t structure for configuration:

  • Output directory settings
  • Threading configuration
  • Cache management
  • Training parameters

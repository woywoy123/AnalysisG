Introduction

The dataloader class manages efficient data loading, batching, and k-fold cross-validation for machine learning training and evaluation. It handles graph data transfer between CPU and GPU devices.

Features

  • K-fold Cross-Validation: Automatic data splitting for cross-validation
  • Train/Test Splitting: Configurable train/test split ratios
  • Batch Construction: Efficient batching of graph data
  • GPU Data Transfer: Automatic tensor transfer to CUDA devices
  • Graph Serialization: Save/restore graph datasets to/from HDF5
  • CUDA Memory Server: Background memory management for GPUs

Batching Details

The build_batch() method constructs batched graphs by:

  1. Collecting Graphs: Gather N graphs for the batch
  2. Concatenating Nodes: Stack all node features [Σn_i, F]
  3. Adjusting Edge Indices: Offset edge indices for batched graph
  4. Creating Batch Index: Track which nodes belong to which graph
  5. Stacking Graph Features: Combine graph-level features [N, G]

The resulting batched graph is compatible with PyTorch Geometric conventions.

K-Fold Cross-Validation

The dataloader supports stratified k-fold cross-validation:

dataloader dl;
// Generate 5-fold splits with 80% train, 20% test
dl.generate_test_set(20); // 20% for final test
dl.generate_kfold_set(5); // 5 folds from remaining 80%
for (int k = 0; k < 5; ++k) {
auto* train = dl.get_k_train_set(k);
auto* valid = dl.get_k_validation_set(k);
// Train on train, validate on valid
}
auto* test = dl.get_test_set();
// Final evaluation on test

Graph Serialization

Graphs can be serialized to HDF5 for faster subsequent loading:

dataloader dl;
// ... populate with graphs ...
// Save to disk
dl.dump_graphs("./graph_cache/", 10); // 10 threads
// Later: restore from disk
dataloader dl2;
dl2.restore_graphs("./graph_cache/", 10);

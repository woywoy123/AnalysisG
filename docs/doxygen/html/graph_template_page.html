Graph Template Guide

Detailed guide for using and extending graph_template.

Introduction

The graph_template class is used to construct graph representations from physics events. It defines how particles become nodes, how edges are constructed, and what features are attached.

Workflow

  1. Receive an event via build(event_template*)
  2. Define nodes using define_particle_nodes()
  3. Define edges using define_topology()
  4. Add features in CompileEvent()
  5. Export to graph_t structure for ML

Example

class MyGraph : public graph_template {
public:
MyGraph() {
// Define edge topology - here: fully connected
define_topology([](particle_template* p1, particle_template* p2) {
return true; // Connect all nodes
});
}
void CompileEvent() override {
// Get the event
MyEvent* ev = get_event<MyEvent>();
// Add graph-level features
add_graph_data_feature<double, MyEvent>(ev,
[](MyEvent* e) { return e->met; }, "met");
// Add node-level features (for each particle)
add_node_data_feature<double, Electron>(
[](Electron* e) { return e->pt; }, "pt");
// Add edge-level features (for each particle pair)
add_edge_data_feature<double, Electron>(
[](std::tuple<Electron*, Electron*>* pair) {
return deltaR(std::get<0>(*pair), std::get<1>(*pair));
}, "deltaR");
// Add truth labels
add_node_truth_feature<int, Electron>(
[](Electron* e) { return e->isFromTop ? 1 : 0; }, "from_top");
}
bool PreSelection() override {
// Optional: skip graphs that don't pass criteria
return num_nodes >= 2;
}
};

Topology

The topology function defines edge connections:

define_topology([](particle_template* p1, particle_template* p2) {
// Return true if p1 and p2 should be connected
// Example: only connect different particles
return p1 != p2;
});

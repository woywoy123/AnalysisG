Model Template Guide

Detailed guide for using and extending model_template.

Introduction

The model_template class provides the base interface for machine learning models in AnalysisG. It integrates with PyTorch (LibTorch) for neural network functionality.

Structure

A model template defines:

  • Network architecture
  • Forward pass logic
  • Input/output feature mapping

Example

class MyModel : public model_template {
public:
torch::nn::Linear fc1, fc2, fc3;
MyModel() {
fc1 = register_module("fc1", torch::nn::Linear(10, 64));
fc2 = register_module("fc2", torch::nn::Linear(64, 64));
fc3 = register_module("fc3", torch::nn::Linear(64, 1));
}
torch::Tensor forward(graph_t* data) override {
auto x = *data->get_data_node("features", this);
x = torch::relu(fc1->forward(x));
x = torch::relu(fc2->forward(x));
x = torch::sigmoid(fc3->forward(x));
return x;
}
};

Training

Models are trained using the optimizer with parameters:

optimizer_params_t params;
params.learning_rate = 0.001;
params.epochs = 100;
params.batch_size = 64;
analysis ana;
ana.add_model(new MyModel(), &params, "training_run");
